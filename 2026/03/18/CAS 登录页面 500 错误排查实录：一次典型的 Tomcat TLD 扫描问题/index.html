六、经验总结

6.1 问题特点

这类问题有以下特征：

  1. 错误信息指向标签库：uri cannot be resolved
  2. JAR 文件存在但无法访问：文件在 WAR 包中，但 Tomcat 扫描不到
  3. 本地开发环境正常：因为开发环境通常使用 IDE 内置的 Tomcat，配置与生产环境不同
  4. Docker 容器环境复现：容器化的 Tomcat 配置可能与标准安装不同

6.2 排查思路

  1. 确认依赖存在：检查 WAR 包中是否包含必要的 JAR 文件
  2. 确认资源可访问：检查 JAR 文件内的 TLD 等资源是否存在
  3. 检查容器配置：检查 Tomcat 的 catalina.properties 等配置文件
  4. 查看完整日志：有时关键信息在日志的其他部分

6.3 预防措施

  1. 标准化 Docker 镜像：使用官方基础镜像，避免自定义配置导致的问题
  2. 配置即代码：将 Tomcat 配置文件纳入版本控制
  3. 环境一致性：确保开发、测试、生产环境的 Tomcat 配置一致
  4. 构建验证：在 CI/CD 流程中添加 WAR 包内容验证步骤

七、参考资源

  • Tomcat 9 官方文档 - TLD 扫描
  • Spring Framework JSP 标签库文档
  • CAS 4.2 官方文档

后记：这个问题看似简单，但涉及 Tomcat 内部机制、Maven 构建配置、Docker 容器化等多个层面。希望这篇排查记录能帮助遇到类似问题的同学少走弯路。
